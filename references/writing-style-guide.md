# Writing Style Guide

This guide defines writing style requirements to ensure generated content sounds natural and human-written, avoiding AI-generated patterns.

---

## Writing Style Quick Reference

| 风格代码 | 风格名称 | 特点 | 适用场景 | 段落长度 | AI味 |
|---------|---------|------|---------|---------|------|
| narrative-deep | 叙述深度型 | 人性化、有深度、有故事 | 深度文章、品牌内容 | 100-150词 | 低 |
| concise-bullet | 简洁列表型 | 简洁、直接、结构化 | 快速阅读、操作指南 | 30-50词 | 高 |
| formal | 正式商务型 | 专业、数据驱动 | 商业报告、白皮书 | 150-200词 | 低 |
| casual | 轻松对话型 | 亲切、易懂 | 博客、社交媒体 | 80-120词 | 低 |
| technical | 技术深度型 | 精确、详细 | 技术文档、教程 | 120-180词 | 中 |

## Core Principle: Natural Human Writing

**Goal**: Content should read as if written by an experienced human writer, not an AI assistant.

**Key Characteristics**:
- Varied sentence structures and lengths
- Natural paragraph flow with transitions
- Narrative depth with examples and context
- Conversational yet professional tone
- Organic idea development

**Note**: This principle applies primarily to `narrative-deep` style. Other styles like `concise-bullet` intentionally use AI-like patterns for efficiency.

## Anti-AI Patterns to Avoid

### ❌ Pattern 1: Short Sentence Stacking

**Problem**: Consecutive short sentences create robotic, choppy rhythm.

**Bad Example**:
```
多渠道增加曝光。
不同平台有不同用户。
AI会发现更多内容。
```

**Good Example**:
```
多渠道分发策略能够显著提升内容的可见性。当你在LinkedIn、Medium、知乎等平台
同步发布时，不同平台的用户群体和AI爬虫都能发现你的内容，从而大幅增加被引用
的机会。这就像在多个路口设置广告牌，而不是只在一个地方等待顾客路过。
```

### ❌ Pattern 2: Bullet-Point Paragraphs

**Problem**: Every paragraph is just a single sentence followed by bullets.

**Bad Example**:
```
### 原因1：扩大覆盖
单一渠道限制了内容的可见性。

### 原因2：多样化信号
不同平台的内容被引用。
```

**Good Example**:
```
### 原因1：扩大覆盖范围

单一渠道的内容分发策略存在明显的局限性。当你的内容只发布在自有网站上时，
AI搜索引擎可能因为爬取频率、权重评估等因素而错过你的优质内容。通过多渠道
分发，你可以在LinkedIn、Medium、知乎等不同平台建立内容据点，大幅增加被AI
发现和引用的机会。

根据我们的观察，采用多渠道策略的企业，其内容被AI引用的概率提升了3-5倍。
这不仅仅是数量上的增加，更重要的是不同平台的引用会相互强化，形成权威性
的累积效应。
```

### ❌ Pattern 3: Lack of Transitions

**Problem**: Paragraphs and sections jump abruptly without connecting phrases.

**Bad Example**:
```
多渠道分发很重要。

LinkedIn是专业平台。
```

**Good Example**:
```
多渠道分发策略的重要性不言而喻。那么，在众多平台中，我们应该如何选择呢？

首先来看LinkedIn。作为全球最大的职业社交平台，LinkedIn拥有独特的优势...
```

### ❌ Pattern 4: Excessive Code Blocks

**Problem**: Over-reliance on code blocks for simple lists.

**Bad Example**:
```
优势：
```
- 完全控制
- SEO优化
- 品牌建设
```
```

**Good Example**:
```
自有网站的优势在于完全控制。你可以自由决定内容的呈现方式、优化策略和用户
体验，而不受第三方平台规则的限制。同时，自有网站也是SEO优化的最佳阵地，
能够长期积累搜索引擎权重。更重要的是，它是品牌建设的核心载体，所有其他
渠道的内容最终都应该引导用户回到这里。
```

## Writing Style Requirements

### 1. Paragraph Structure

**Minimum Requirements**:
- Each paragraph: **3-8 sentences**
- Minimum paragraph length: **80 words**
- Avoid single-sentence paragraphs (except for emphasis)

**Composition**:
- **60% narrative paragraphs** - Flowing text with explanations
- **40% structured content** - Lists, code blocks, tables

**Example Structure**:
```
[Topic sentence introducing the concept]
[Explanation with context and background]
[Example or data supporting the point]
[Implication or application]
[Transition to next idea]
```

### 2. Sentence Variety

**Length Distribution**:
- Short sentences (5-10 words): 20%
- Medium sentences (11-20 words): 50%
- Long sentences (21-30 words): 30%

**Structure Mix**:
- Simple sentences: 40%
- Compound sentences: 35%
- Complex sentences: 25%

**Avoid**:
- ❌ 3+ consecutive short sentences
- ❌ All sentences starting the same way
- ❌ Repetitive sentence patterns

**Example of Good Variety**:
```
GEO优化需要长期投入。(短句，9词)

但这种投入是值得的，因为一旦建立起权威性，你的内容就会持续为你带来价值。
(中句，18词)

与付费广告的即时效果不同，GEO更像是种树而非买花——虽然需要时间培育，但
最终会长成参天大树，为你遮风挡雨，而不是花谢之后就一无所有。(长句，28词)
```

### 3. Content Depth

**Every Key Point Must Include**:
1. **Context** - Why this matters
2. **Explanation** - How it works
3. **Example** - Real-world application
4. **Data** (when possible) - Numbers or evidence
5. **Actionable insight** - What to do

**Bad Example** (Shallow):
```
### 使用Schema标记

Schema标记很重要。
它帮助AI理解内容。
应该添加到网站。
```

**Good Example** (Deep):
```
### 使用Schema标记提升AI理解

Schema标记是一种结构化数据格式，它就像给你的内容贴上了"营养成分表"，
让AI能够快速准确地理解内容的类型、作者、发布时间等关键信息。

为什么这很重要？当AI面对海量网页时，Schema标记能让它在几毫秒内判断你的
内容是否相关，而不需要"阅读"整篇文章。这就像超市里的商品标签——顾客
不需要打开每个包装就能知道里面是什么。

根据Google的研究，添加了Schema标记的页面，其在搜索结果中的点击率平均
提升30%。对于AI搜索引擎来说，这个效果可能更加显著，因为AI更依赖结构化
数据来做决策。

实施建议：优先为文章添加Article Schema，包括标题、作者、发布时间、修改
时间等基本信息。如果有FAQ内容，务必添加FAQPage Schema，这是AI最喜欢
引用的内容类型之一。
```

### 4. Natural Transitions

**Use Transition Phrases**:
- 因果关系: "因此"、"所以"、"正因如此"
- 对比关系: "相比之下"、"与此不同"、"另一方面"
- 递进关系: "更重要的是"、"不仅如此"、"进一步来说"
- 举例说明: "例如"、"比如说"、"具体来说"
- 总结归纳: "总的来说"、"综上所述"、"简而言之"

**Example**:
```
多渠道分发能够扩大内容覆盖范围。[观点1]

更重要的是，[过渡] 不同平台的引用会相互强化。当AI在多个权威平台上
发现相同的内容时，它会认为这个内容更加可信。[观点2]

然而，[转折] 这并不意味着要盲目追求平台数量。相比之下，[对比] 选择
3-5个与目标受众高度匹配的平台，深度运营，往往比在10个平台上浅尝辄止
更有效。[观点3]
```

### 5. Storytelling Elements

**Use Analogies and Metaphors**:
```
❌ 弱: "GEO需要长期投入"
✅ 强: "GEO就像种树而非买花——需要时间培育，但最终会长成参天大树"
```

**Use Scenarios**:
```
想象一下这样的场景：一位潜在客户向ChatGPT询问"如何选择项目管理工具"。
如果你的内容足够优质、结构化程度高、权威性强，ChatGPT就会引用你的文章，
甚至直接推荐你的产品。这就是GEO的魅力所在。
```

**Use Concrete Examples**:
```
❌ 抽象: "某公司通过GEO获得了成功"
✅ 具体: "HubSpot通过发布300+篇深度营销指南，成为AI搜索'营销自动化'
话题的首选引用源，每月从AI引用获得10万+访问量"
```

### 6. Conversational Tone

**Use Questions**:
```
那么，如何判断你的内容是否适合多渠道分发呢？
```

**Use Direct Address** (Sparingly):
```
你可能会问："这需要多少时间投入？"
```

**Use Informal Connectors**:
```
说到这里，我们不得不提...
这里有个有趣的现象...
让我们来看一个实际案例...
```

## Style-Specific Guidelines

### narrative-deep Style (Recommended for Deep Content)

**Formerly known as**: soul-scribe-c

**Characteristics**: Human-like writing with depth, storytelling, and natural flow.

**Structure**:
1. **核心要点速览表** - Quick reference table
2. **场景化开头** - Start with a scenario or question (100-150 words)
3. **深度分析** - 3-8 paragraphs per section
4. **实施清单** - Actionable checklist
5. **立即行动** - 3-4 immediate next steps

**Paragraph Requirements**:
- Average: 100-150 words per paragraph
- Mix narrative (60%) with structured content (40%)
- Include at least 1 example or analogy per major section
- 3-8 sentences per paragraph

**Tone**:
- Professional yet conversational
- Data-driven but accessible
- Practical and actionable

**Example Opening**:
```markdown
# 内容分发：多渠道GEO策略

**副标题：扩大AI可见性的范围**

## 核心要点速览
[表格]

---

想象一下这样的场景：你花了一周时间精心打磨了一篇5000字的深度文章，
发布在自己的网站上，然后就坐等AI引用。一个月过去了，引用寥寥无几。
问题出在哪里？

答案可能不是内容质量，而是可见性。在AI搜索的世界里，"酒香也怕巷子深"。
即使你的内容再优质，如果AI搜索引擎找不到它，或者认为它的权威性不足，
就不会引用。这就是为什么多渠道分发策略如此重要。

根据我们对100家企业的GEO实践研究，采用多渠道分发策略的企业，其内容被AI
引用的概率是单一渠道的3-5倍。更重要的是，不同平台的引用会相互强化，
形成权威性的累积效应。本文将详细解析如何构建高效的多渠道GEO分发策略。
```

### concise-bullet Style (Fast Reading, AI-Style)

**Characteristics**: Concise, structured, list-heavy content optimized for quick scanning.

**Structure**:
1. **核心要点速览表** - Quick reference table
2. **简短引入** - 1-2 sentences introduction
3. **列表式内容** - Heavy use of bullets and code blocks
4. **实施清单** - Actionable checklist
5. **立即行动** - 3-4 immediate next steps

**Paragraph Requirements**:
- Average: 30-50 words per paragraph
- 80% lists/code blocks, 20% narrative
- 1-3 sentences per paragraph
- Minimal storytelling

**Tone**:
- Direct and efficient
- Information-dense
- Action-oriented

**Example Opening**:
```markdown
# 内容分发：多渠道GEO策略

**副标题：扩大AI可见性的范围**

## 核心要点速览
[表格]

---

## 为什么需要多渠道分发

### 原因1：扩大覆盖
单一渠道限制了内容的可见性。

**优势**：
- 多平台曝光
- 增加发现机会
- 降低风险

### 原因2：多样化信号
不同平台的内容被引用，增强权威性。

**效果**：
- 提升可信度
- 强化权威性
- 增加引用率
```

**Use Cases**:
- 操作指南和步骤说明
- 快速参考文档
- 技术配置说明
- 需要快速浏览的内容

---

### formal Style

**Requirements**:
- Longer paragraphs (150-200 words)
- More data and citations
- Executive summary at beginning
- Formal transitions
- Professional terminology

### casual Style

**Requirements**:
- Shorter paragraphs (80-120 words)
- More personal anecdotes
- Conversational questions
- Informal language
- Relatable examples

### technical Style

**Requirements**:
- Technical depth with code examples
- Precise terminology
- Architecture explanations
- Implementation details
- Still maintain narrative flow (not just code dumps)

## Quality Checklist

Before finalizing any article, verify:

### Paragraph Quality
- [ ] No single-sentence paragraphs (except for emphasis)
- [ ] Each paragraph has 3-8 sentences
- [ ] Each paragraph is 80+ words
- [ ] 60% narrative, 40% structured content

### Sentence Quality
- [ ] Varied sentence lengths (short/medium/long mix)
- [ ] No 3+ consecutive short sentences
- [ ] Mix of simple, compound, and complex sentences
- [ ] Natural rhythm when read aloud

### Content Depth
- [ ] Every key point has context + explanation + example
- [ ] Includes specific data or evidence
- [ ] Provides actionable insights
- [ ] Uses analogies or metaphors

### Flow and Transitions
- [ ] Smooth transitions between paragraphs
- [ ] Logical progression of ideas
- [ ] Clear connections between sections
- [ ] Natural reading flow

### Human Touch
- [ ] Conversational tone (not robotic)
- [ ] Includes scenarios or stories
- [ ] Uses concrete examples (not abstract)
- [ ] Feels like human-written content

## Implementation in Content Generation

When generating content, Claude should:

1. **Read this guide** before starting generation
2. **Apply requirements** throughout the writing process
3. **Self-check** against the quality checklist
4. **Revise** any sections that feel robotic or AI-generated
5. **Ensure** natural flow and human-like writing

**Key Reminder**: The goal is not just to follow rules, but to create content that genuinely sounds like it was written by an experienced human writer who understands the topic deeply and can explain it clearly and engagingly.
